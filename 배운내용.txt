### KG_P1
- Cypher 기본 문법
    - 가변길이 경로 검색
    - 최단경로
    - 집계함수 
    - 제약조건/인덱스
    - csv 데이터 -> 지식 그래프 변환 (csv를 파악하여 노드로 변환하는 것이 수작업이였음, 범용적 x)
    - 추천 쿼리에서 가중치를 주는법 (단순 곱셈이였음)
    
- 전문 검색 (Full-Text Search) / 전문 검색 인덱스 설정 
    - 기본 전문 검색 
    - 퍼지 검색
    - 와일드카드 검색
    - 정확한 구문 검색
    - 논리연산자 검색 (AND/OR/NOT)
    - 가중치부여 검색
    - 응용 -> 전문검색 후 그래프 탐색
    - 두 가지 전문 인덱스를 모두 사용하는 복합 검색 (두 서브 쿼리의 결과를 연결하여 조건을 걸어필터링 하는 법) 
      Feat. 이름없는 서브쿼리(이름 없는 호출)

- 그래프 DB에서 벡터 사용
    - ★★★백터 인덱스 생성 
    - 기존에 있는 노드의 속성을 추출하여 임베딩하여 벡터 속성 추가하기
    - ★★★ 벡터 검색 (자연어 -> 임베딩 -> 벡터값으로 DB에 직접 프로시저를 이용하여 유사도 검색)
    - ★★★응용 -> 벡터 검색 + 키워드 필터링 (filter 조건을 계속 추가할 수 있는 구조를 배움)    

- Text-to-Cypher 
        - 1. 쿼리 결과만 반환하는 방법과 2. 쿼리 결과를 바탕으로 LLM이 응답을 생성하는 방법이 있음
        - 쿼리용 LLM과, 최종 답변 생성용 LLM을 따로 설정할 수 있음 
        - Cypher 생성을 위한 특화 프롬프트 만들어서 적용하는 법

- GraphRAG
    - 백터검색(자연어로만, 유사도 기준)
    - MMR 알고리즘을 활용한 다양성 검색 (유사도 + 다양성)
    - 응용 -> 백터검색(자연어로만) + 그래프 검색
    - LCEL을 사용하여 RAG 체인 구성 (전에도 LCEL을 들어봤는데 그 당시에 이해못했다가 여기서 이해하게됨)
    - Graph DB 기반 Retriever 활용 
        - 벡터 검색(벡터 or 자연어) + 증강검색 // 자연어로 검색하는게 편했음
